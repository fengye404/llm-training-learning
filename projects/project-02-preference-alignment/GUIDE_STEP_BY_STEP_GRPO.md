# 第 3 章讲义模式（GRPO）：10 行一讲

配合文件：`projects/project-02-preference-alignment/grpo_train.py`

```mermaid
flowchart LR
    A[同一问题多个候选] --> B[给每个候选奖励]
    B --> C[算组平均奖励]
    C --> D[advantage=奖励-组均值]
    D --> E[更新 logits]
    E --> F[概率向高优势回答集中]
```

## 第 1 段（1-12 行）

白话解释：
- 文件头说清楚：GRPO 是做“组内相对比较”。

练习：
- 用你的话解释“组内比较”是什么意思。

## 第 2 段（14-25 行）

白话解释：
- `GROUPS` 里每个问题都有多个候选和对应奖励。
- 奖励高代表更优回答。

练习：
- 给第一个问题新增一个候选并给奖励。

## 第 3 段（28-34 行）

白话解释：
- `softmax` 把 logits 变成概率。
- 未来概率会往高奖励候选上集中。

练习：
- 临时打印 `probs` 并检查总和约等于 1。

## 第 4 段（37-45 行）

白话解释：
- 初始化超参数和每个问题的 logits（全 0 起步）。

练习：
- 把 `epochs` 改为 `10`，看最终概率是否已经明显偏向好候选。

## 第 5 段（47-61 行）

白话解释：
- 每轮先算 `probs`、`rewards`、`avg_reward`。
- `advantages = reward - avg_reward` 是核心。

练习：
- 打印 `advantages`，观察谁为正谁为负。

## 第 6 段（63-68 行）

白话解释：
- 目标函数 `obj` 会鼓励高优势回答的概率上升。

练习：
- 观察 `objective` 是否整体上升（越大越好）。

## 第 7 段（70-77 行）

白话解释：
- 参数更新：根据优势值调整 logits。
- 高优势候选 logits 增长更快。

练习：
- 把 `lr` 改成 `0.1` 和 `0.8` 对比结果。

## 第 8 段（79-87 行）

白话解释：
- 打印最终候选概率，验证好回答是否接近 1。

练习：
- 把一条好回答的奖励调低，观察最终概率如何变化。

## 第 9 段（90-91 行）

白话解释：
- Python 入口。

练习：
- 解释为什么脚本要有入口判断。

## 过关标准

1. 你能解释 advantage 的直觉。
2. 你能解释“相对好”而不是“绝对好”的意义。
3. 你能通过奖励改动预测概率变化方向。
