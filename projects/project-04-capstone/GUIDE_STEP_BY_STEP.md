# 第 5 章讲义模式：自动生成学习报告（10 行一讲）

配合文件：`projects/project-04-capstone/build_learning_report.py`

```mermaid
flowchart LR
    A[实验指标列表] --> B[计算 delta]
    B --> C[拼接 Markdown]
    C --> D[写入文件]
```

## 第 1 段（1-13 行）

白话解释：
- 文件头说明目标：把实验结果变成报告。
- 导入 `dataclass` 和日期工具。

练习：
- 用一句话说明为什么“自动生成报告”有价值。

## 第 2 段（15-30 行）

白话解释：
- 定义 `Experiment`，是单条实验记录模型。
- 相当于 Java DTO。

练习：
- 新增一个字段（如 `owner`）并思考报告中怎么展示。

## 第 3 段（33-40 行）

白话解释：
- `EXPERIMENTS` 是示例数据。
- 每条含“变更前/后”，便于算差值。

练习：
- 加入一条你自己的实验记录。

## 第 4 段（43-53 行）

白话解释：
- `to_markdown` 开始构建 Markdown 文本头和表头。

练习：
- 把报告标题改成“我的训练周报”。

## 第 5 段（54-58 行）

白话解释：
- 遍历每条实验记录，计算 delta 并写入表格行。

练习：
- 把 `delta` 保留两位小数，观察表格变化。

## 第 6 段（60-66 行）

白话解释：
- 拼接“下一步行动”，形成完整报告文本。

练习：
- 改成你这周真实要做的 3 条行动。

## 第 7 段（69-78 行）

白话解释：
- `main` 调用生成函数并写入 `learning_report.md`。

练习：
- 把输出文件名改成 `weekly_report.md`。

## 第 8 段（81-82 行）

白话解释：
- Python 入口。

练习：
- 解释为什么要入口判断而不是直接执行。

## 过关标准

1. 你能把模拟数据替换成真实数据。
2. 你能改报告模板并输出新文件。
3. 你能每周稳定产出一版报告。
